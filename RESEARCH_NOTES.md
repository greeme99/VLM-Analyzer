# VLM MODAPTS Motion Analyzer - 기술 조사 결과

## 1. MODAPTS 표준 코드 체계

### 1.1 기본 단위: MOD (Modular Unit Definition)
- **1 MOD = 0.129 seconds** (표준 시간 단위)
- 모든 코드는 MOD의 배수로 표현
- 예: 10 MODs = 10 × 0.129 = 1.29 seconds

### 1.2 표준 코드 목록

#### Move (M) - 손/팔 이동
| 코드 | 설명 | MOD | 초 |
|------|------|-----|-----|
| M1 | 손가락 이동 (2.5cm/1") | 1 | 0.129 |
| M2 | 손 이동 (5cm/2") | 2 | 0.258 |
| M3 | 전완 이동 (15cm/6") | 3 | 0.387 |
| M4 | 팔 이동 (30cm/12") | 4 | 0.516 |
| M5 | 팔 + 몸통 이동 (60cm/24") | 5 | 0.645 |
| M6 | 팔 + 몸통 + 다리 이동 (90cm+) | 6 | 0.774 |

#### Get (G) - 물체 집기/쥐기
| 코드 | 설명 | MOD | 초 |
|------|------|-----|-----|
| G0 | 손가락으로 집기 (쉬운 물체) | 0 | 0 |
| G1 | 손으로 집기 (일반 물체) | 1 | 0.129 |
| G2 | 양손으로 집기 (무거운 물체) | 2 | 0.258 |
| G3 | 도구 사용 집기 | 3 | 0.387 |

#### Place (P) - 물체 놓기
| 코드 | 설명 | MOD | 초 |
|------|------|-----|-----|
| P0 | 가볍게 놓기 | 0 | 0 |
| P1 | 일반 놓기 | 1 | 0.129 |
| P2 | 조심스럽게 놓기 | 2 | 0.258 |

#### 기타 코드
| 코드 | 설명 | MOD |
|------|------|-----|
| D3 | 결정/판단 | 3 |
| L1 | 읽기 (짧은 텍스트) | 1 |
| E2 | 눈 이동 | 2 |
| F3 | 발 이동 | 3 |
| R2 | 읽기 (일반) | 2 |

### 1.3 MODAPTS 분석 프로세스
1. 작업을 기본 동작 요소로 분해
2. 각 동작에 해당하는 코드 부여
3. 각 코드의 MOD 값 합산
4. 총 MOD × 0.129 = 표준시간(초)

## 2. VLM 모델 선택 및 활용

### 2.1 GPT-4V (권장)
**장점:**
- 높은 정확도의 동작 인식
- 복잡한 프롬프트 제어 가능
- JSON 형식 출력 유도 용이
- 다양한 각도/조명 환경에서 강건함

**단점:**
- API 비용 발생 (이미지당 약 $0.01-0.03)
- 인터넷 연결 필수

**활용 방식:**
- 프레임을 Base64 인코딩하여 API 전송
- 프롬프트: "산업공학 전문가 역할로 MODAPTS 코드 부여"
- 응답: JSON 형식 (start_time, end_time, code, description)

### 2.2 LLaVA-NeXT-Video (오픈소스 대안)
**장점:**
- 무료 (로컬 배포 가능)
- 비디오 직접 입력 지원
- 완전 오픈소스

**단점:**
- 로컬 GPU 필요 (최소 8GB VRAM)
- 정확도가 GPT-4V보다 낮을 수 있음

## 3. 프레임 분석 파이프라인 기술

### 3.1 손 추적 (Mediapipe)
- 21개 랜드마크 감지 (손가락 끝, 관절)
- 실시간 처리 가능
- 신뢰도 점수 함께 제공

### 3.2 부품 객체 탐지
- 간단한 컬러 기반 추적 (초기 버전)
- YOLO v8 (고급 버전)
- 손과 부품 사이 거리 계산

### 3.3 동작 구간 분해
- **Reach (M)**: 손이 부품에서 멀 때
- **Get (G)**: 손이 부품에 닿을 때
- **Place (P)**: 손이 부품에서 떨어질 때
- 거리 임계값 기반 상태 전환

## 4. 구현 기술 스택 최종 결정

### 백엔드
- **프레임 처리**: OpenCV (Python)
- **손 추적**: Mediapipe (Python)
- **VLM 분석**: OpenAI GPT-4V API
- **API 서버**: FastAPI (Python)
- **데이터베이스**: MySQL (Manus 제공)

### 프론트엔드
- **프레임워크**: React 19 + Tailwind CSS 4
- **상태 관리**: tRPC + React Query
- **차트**: Recharts (다크 테마 최적화)
- **파일 업로드**: S3 스토리지
- **다국어**: i18next

### 배포
- **서버**: Express.js (Node.js)
- **스토리지**: S3 (Manus 제공)
- **호스팅**: Manus 플랫폼

## 5. 주요 구현 고려사항

### 5.1 성능 최적화
- 프레임 샘플링: 1-2 FPS (초기), 필요시 6-15 FPS로 조정
- 배치 처리: 여러 프레임을 한 번에 VLM에 전송
- 캐싱: 분석 결과 DB 저장으로 중복 분석 방지

### 5.2 정확도 향상
- Few-shot 프롬프트: 예제 포함으로 VLM 판정 일관성 향상
- 수동 보정 UI: 사용자가 잘못된 코드 수정 가능
- 모델 앙상블: GPT-4V + 로컬 모델 결합

### 5.3 사용자 경험
- 진행률 표시: 분석 중 실시간 상태 업데이트
- 드래그앤드롭: 직관적인 파일 업로드
- 대시보드: 시각적 데이터 표현 (차트, 테이블)
- 보정 UI: 간편한 코드 수정 및 코멘트 입력
